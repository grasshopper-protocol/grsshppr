# Design Principles

The source of truth for how Grasshopper looks and behaves. Distilled from
`DESIGN.md` (repo root), which holds the fuller visual brief.

1. **Every element earns its place.** If removing it doesn't break anything,
   remove it.
2. **Quiet confidence over loud persuasion.** No exclamation marks in the UI.
   No "amazing!", no confetti.
3. **Content-first hierarchy.** The mentor's expertise and the mentee's goals
   are the hero — not the chrome around them.
4. **Accessibility is non-negotiable.** WCAG 2.1 AA minimum. Focus states,
   contrast ratios, screen-reader support. Beautiful and accessible are not in
   tension.
5. **Progressive density.** New users see less; committed pairs unlock more
   (notes, goals). Complexity is earned.

**References:** Linear, Vercel, Raycast, Stripe — expensive without being loud.
**Anti-references:** ADPList (too playful), Dribbble maximalism, corporate SaaS
dashboards.

> For color, typography, spacing, motion, IA, and UI copy tone, see `DESIGN.md`.
> These will move into `tokens/` and `patterns/` as the system is formalised.
